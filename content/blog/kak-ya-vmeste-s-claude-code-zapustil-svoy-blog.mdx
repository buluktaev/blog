Продуктовый дизайнер может запустить полноценный блог с автоматизированным пайплайном, не написав ни строчки кода. Достаточно Claude Code Skills, Next.js 15 и точных промптов — агент берёт на себя стек, деплой и форматирование, а ты занимаешься текстом. Этот пост — доказательство: он создан, отформатирован и опубликован через тот самый пайплайн, о котором в нём рассказывается.

## Сорок заметок в Notion и ни одного поста

За несколько месяцев я накопил больше сорока заметок о работе с AI-агентами. Наблюдения о дизайн-процессе, о том, как продуктовое мышление меняется, когда рядом есть Claude. Черновиков хватало. Публикаций — ноль.

Меня зовут Санан, я продуктовый дизайнер. Работаю в рекрутинговом агентстве над внутренней ATS-системой для найма IT-специалистов. До этого строил фичи в стартапах: Quiz Lab — образовательные квизы, и Sowaa — умный матрас с анализом фаз сна и мобильным приложением. Я не программист. Мой инструмент — промпт, не IDE.

И вот передо мной лежала конкретная пропасть между «есть черновик» и «пост опубликован»:

- Выбор стека и шаблона
- Настройка деплоя
- Форматирование MDX с правильным frontmatter
- Git-коммиты, пуши, CI/CD
- И самое болезненное — вопрос голоса: как не потерять себя, когда AI помогает с текстом

Две задачи одновременно: убрать техническое трение и сохранить голос. Я решил обе промптами.

## Боль: «Опиши, каким должен быть блог»

Первый промпт был наивным. Я попросил агента «сделать блог на Astro Paper». Агент склонировал репозиторий, настроил конфиг, статика заработала. Lighthouse — 100. Деплой на Vercel — без ручной CI-конфигурации.

Но через неделю я уже мигрировал на Next.js 15. Причина прозаичная: мне нужны были MDX-компоненты — интерактивные блоки внутри постов, кастомные layouts, гибкость, которую статический генератор не давал. Попросил агента перенести контент — он перенёс за один сеанс. Все посты, frontmatter, стили. Блог сейчас работает на Next.js 15 + Vercel.

Ключевое: я ни разу не открывал IDE. Описал — агент сделал. Описал точнее — агент переделал.

## Промпт: скилл для контент-пайплайна

Claude Code Skills — механизм, доступный с декабря 2025. Файл `SKILL.md` с progressive disclosure: при старте агент видит ~100 токенов — короткое описание. При вызове `/blog-post` разворачивается полная инструкция — ~2 000 токенов с шаблонами, правилами, примерами.

Я собрал скилл для контент-пайплайна. Вот как он работает:

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  Черновик    │────▶│  Claude Code  │────▶│  MDX        │
│  (заметки)  │     │  Skill        │     │  + front-   │
│             │     │  /blog-post   │     │  matter     │
└─────────────┘     └──────┬───────┘     └──────┬──────┘
                           │                     │
                    ┌──────▼───────┐     ┌──────▼──────┐
                    │  Проверка    │     │  git commit  │
                    │  голоса и    │     │  + push      │
                    │  структуры   │     │  → Vercel    │
                    └──────────────┘     └─────────────┘
```

Одна команда — агент берёт черновик, структурирует по плану, проверяет тон, генерирует frontmatter, коммитит и пушит. Vercel подхватывает — пост в сети.

## Реакция агента: первый блин и три итерации

Первая версия скилла была слишком свободной. Агент структурировал текст, но добавлял от себя — «подводя итог», «стоит подчеркнуть». Корпоративный тон пролезал в каждый абзац. Я увидел проблему и переписал промпт.

Вторая итерация — добавил replacement patterns[^1] прямо в `SKILL.md`. Пары вида «если агент написал X → замени на Y». Семнадцать пар. «Написал код» → «попросил агента». «Стоит отметить» → удалить. Менторский тон → отсечь. Агент стал послушнее, но голос всё ещё был ровным — предложения одной длины, никакой дерзости.

Третья итерация — добавил writing guide с конкретными правилами ритма: «короткие утверждения чередуются со сложными конструкциями», «em-dash для пауз», «юмор прячется в скобках (вот так)». Вот тогда текст зазвучал.

Три принципа, которые я вынес из этих итераций:

**Агент — инструмент, не автор.** AI не придумывает, о чём мне писать. Я прихожу с заметками, тезисами, конкретными фактами. Агент структурирует, форматирует, проверяет. Разница — как между «нарисовать картину» и «натянуть холст на подрамник».

**Голос фиксируется в конфиге, а не в надежде.** В скилле прописан writing guide — синтаксические паттерны, антипаттерны, replacement patterns. Если агент пишет «стоит подчеркнуть» — скилл перехватывает. Голос — система правил, а не случайность.

**Progressive disclosure экономит контекст.** Скилл не загружает 2 000 токенов инструкций при каждом запуске. При старте — 100 токенов. Полная инструкция разворачивается только при вызове `/blog-post`. В длинных сессиях контекстное окно на вес золота.

## Практика: один сеанс от черновика до публикации

Открываю Claude Code. Черновик — сырые мысли, факты, пара цитат. Даю промпт:

> Возьми черновик ниже. Не переписывай — только структурируй по плану:
> [вставить структуру поста].
> Сохрани мой тон: короткие предложения, без корпоративного языка,
> от первого лица. Если чего-то не хватает — спроси, не выдумывай.
>
> ЧЕРНОВИК:
> [...]

«Спроси, не выдумывай» — не вежливость, а часть дизайна промпта. AI-модели склонны к галлюцинациям, особенно когда чувствуют пробел, который можно «заполнить». Явный запрет на выдумку ставит границу.

Агент возвращает структурированный MDX. Я читаю, правлю — обычно 10–15 минут. Потом агент коммитит, пушит. Vercel деплоит. Готово.

Весь цикл — один сеанс. Никакого переключения между инструментами.

## Perplexity, burstiness и зачем формализовать голос

Две метрики оценки «человечности» текста: perplexity (предсказуемость следующего токена) и burstiness (разброс длины предложений).

AI-тексты ровные — средняя длина предложения стабильная, слова предсказуемые. Человеческие тексты рваные. Короткое предложение. Потом длинное, с подчинёнными конструкциями, уточнениями в скобках, внезапной сменой ритма — и снова короткое.

Writing guide в моём скилле намеренно задаёт burstiness. Это инженерное решение для аутентичности, а не стилистический каприз[^2].

Голос не возникает из ниоткуда. Его нужно формализовать. Парадокс: чтобы текст звучал естественно, правила должны быть записаны явно. Иначе каждый новый промпт — лотерея.

## Результат и что из этого следует

Этот пост — результат пайплайна. Lighthouse — 100. От черновика до публикации — один сеанс, не день и не неделя.

Самый частый страх, который я слышу от коллег: «AI будет писать за меня, и это будет не моё». Справедливо — если дать AI пустой промпт и попросить «напиши пост про X». Тогда получишь усреднённый текст, написанный для всех и ни для кого.

Но если ты приходишь с мыслями, фактами, голосом — AI становится мультипликатором. Убирает трение между идеей и публикацией. Берёт на себя стек, форматирование, деплой — всё, что не имеет отношения к тексту, но без чего текст не увидит читатель.

Я дизайнер. Я проектирую опыт — в том числе свой собственный опыт создания контента. Claude Code Skills — способ задизайнить пайплайн так же, как я дизайню интерфейс: с системой, с правилами, с уважением к пользователю. Только пользователь здесь — я сам.

[^1]: Replacement patterns — пары «если агент написал X → замени на Y». Например: «написал код» → «попросил агента написать». Простой механизм, но он радикально меняет ощущение от текста.

[^2]: Чтобы текст не казался написанным AI, нужно дать AI больше инструкций, а не меньше. Контринтуитивно, но работает.
